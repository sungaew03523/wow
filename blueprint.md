
# Blueprint: Редизайн в стиле игрового аукциона

## Обзор

Цель этого проекта — полностью переработать существующее Flutter-приложение, чтобы оно соответствовало дизайну, представленному на изображении. Новый дизайн имеет темную тему, напоминающую пользовательский интерфейс видеоигры, с определенной цветовой палитрой, шрифтами и компоновкой.

## План реализации

### 1. Настройка темы и стилей

- **Цветовая схема:**
  - Основной фон: Очень темный серый (`#1A1A1A`).
  - Фон панелей: Немного светлее, чем основной (`#2C2C2C`).
  - Основной цвет текста/акцентов: Золотисто-желтый (`#FFC700`).
  - Цвет кнопок поиска: Темно-красный фон с золотой рамкой.
- **Типографика:**
  - Будет использован пакет `google_fonts` для применения подходящего шрифта (например, `Roboto` или `Lato`).
  - Текстовые стили будут настроены для заголовков, основного текста и кнопок.
- **Стили кнопок:**
  - Кнопки на боковой панели будут иметь вдавленный вид с темным фоном и золотым текстом.

### 2. Структура и макет

- **Основной экран (`Scaffold`):**
  - **Верхняя панель (`AppBar`):** Будет создана кастомная верхняя панель, содержащая:
    - Выпадающий список (Dropdown).
    - Поле для поиска (`TextField`).
    - Кнопка "Search".
  - **Тело (`Body`):** Будет разделено на две колонки с помощью `Row`:
    - **Левая колонка:** Панель навигации с категориями.
    - **Правая колонка:** Основная область для отображения списка предметов.

### 3. Компоненты UI

- **`CategoryMenu` (Меню категорий):**
  - Виджет для левой панели, отображающий список категорий в виде кнопок.
- **`AuctionList` (Список предметов):**
  - Виджет для правой панели.
  - Будет реализован в виде `DataTable` или кастомного `ListView` для отображения предметов, их цен и доступного количества.
- **Модели данных:**
  - Будут созданы классы для представления данных, такие как `AuctionItem` (предмет аукциона) и `Category` (категория).

### 4. Данные

- На начальном этапе для заполнения интерфейса будут использоваться фиктивные (mock) данные, чтобы точно воссоздать внешний вид.

### 5. Зависимости

- В `pubspec.yaml` будут добавлены следующие пакеты:
  - `google_fonts`: для кастомных шрифтов.
  - `provider`: для управления состоянием темы (хотя в данном случае используется только темная тема, это хорошая практика).
